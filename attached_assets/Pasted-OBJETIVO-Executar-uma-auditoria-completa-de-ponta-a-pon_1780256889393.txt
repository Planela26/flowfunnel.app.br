OBJETIVO: Executar uma auditoria completa de ponta a ponta (E2E) do sistema em ambiente de produção e homologação, identificando falhas críticas antes do lançamento do MVP.

TESTES OBRIGATÓRIOS

1. ISOLAMENTO DE DADOS (MULTI-TENANT)

Verifique se existe qualquer possibilidade de um usuário visualizar, editar ou excluir dados pertencentes a outro usuário.

Testar:

* Usuário A acessando dashboard.
* Usuário B acessando dashboard.
* Alteração manual de IDs nas URLs.
* Alteração manual de IDs nas requisições da API.
* Testes de autenticação e autorização.
* Acesso a métricas, CRM, funis, campanhas, contatos e integrações de outros usuários.

Resultado esperado:

* Nenhum usuário deve visualizar ou manipular dados de terceiros.
* Todas as consultas devem ser filtradas pelo tenant_id/user_id correto.

Gerar relatório detalhado de todas as rotas e endpoints testados.

---

2. TESTE COMPLETO DE TRIAL

Criar uma conta nova.

Verificar:

* Trial é ativado corretamente.
* Data de expiração é registrada corretamente.
* Usuário recebe acesso premium durante o período.
* Trial expira exatamente na data prevista.
* Após expiração o acesso premium é removido.
* Usuário não consegue criar múltiplos trials usando o mesmo e-mail.
* Verificar possíveis brechas para burlar o trial.

Resultado esperado:

* Trial inicia e encerra corretamente.
* Nenhum acesso premium permanece após expiração.

---

3. TESTE DE PAGAMENTOS

Executar fluxo completo utilizando ambiente de testes.

Stripe:

* Criar assinatura.
* Confirmar webhook recebido.
* Confirmar atualização automática do plano.
* Simular renovação.
* Simular cancelamento.
* Simular falha de pagamento.

Mercado Pago:

* Criar assinatura.
* Confirmar webhook recebido.
* Confirmar atualização automática do plano.
* Simular renovação.
* Simular cancelamento.
* Simular falha de pagamento.

Resultado esperado:

* Todos os eventos atualizam o status do usuário corretamente.

---

4. TESTE DE CANCELAMENTO

Criar assinatura ativa.

Verificar:

* Cancelamento imediato.
* Cancelamento ao final do ciclo.
* Mudança automática para plano gratuito.
* Remoção de recursos premium.
* Bloqueio de funcionalidades pagas.

Resultado esperado:

* Usuário não mantém acesso premium após cancelamento.

---
