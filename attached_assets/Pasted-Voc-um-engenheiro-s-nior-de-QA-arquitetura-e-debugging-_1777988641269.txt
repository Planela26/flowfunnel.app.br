Você é um engenheiro sênior de QA, arquitetura e debugging especializado em SaaS B2B multi-tenant.

Sua tarefa é fazer uma AUDITORIA COMPLETA do sistema inteiro, verificando todas as funcionalidades, integrações, fluxos e permissões de cada módulo individualmente.

OBJETIVO:
Garantir que todas as funcionalidades estejam funcionando corretamente para TODOS os clientes da plataforma, sem diferenças entre contas, sem falhas de isolamento de dados e sem bugs ocultos.

EXECUTE UMA VERIFICAÇÃO PROFUNDA EM:

# 1. MULTI-TENANT / CLIENTES

* Verifique se cada cliente possui ambiente isolado.
* Confirme que dados de um cliente NÃO aparecem para outro.
* Verifique permissões, autenticação e separação por tenant.
* Teste criação de múltiplas contas.
* Verifique se todas possuem exatamente as mesmas funcionalidades disponíveis.
* Confirme que planos, permissões e limites funcionam corretamente.
* Teste comportamento com contas novas e contas antigas.

# 2. DASHBOARD

* Verifique carregamento completo.
* Teste widgets, gráficos e métricas.
* Verifique velocidade de troca entre abas.
* Procure lentidão, travamentos ou loops de renderização.
* Analise consumo excessivo de memória.
* Verifique se dados carregam corretamente após refresh.

# 3. INTEGRAÇÕES DE PLATAFORMAS

TESTAR INDIVIDUALMENTE:

* Google Ads
* Facebook Ads
* Instagram Ads
* TikTok Ads
* Stripe
* APIs externas

VERIFICAR:

* OAuth funcionando
* Tokens sendo armazenados corretamente
* Renovação de token
* Expiração de sessão
* Erros de autenticação
* Rate limits
* Sincronização de dados
* Atualização automática
* Dados duplicados
* Dados inconsistentes

# 4. BACKEND

* Verifique todas as rotas da API.
* Procure endpoints expostos indevidamente.
* Verifique autenticação em TODAS as rotas.
* Analise possíveis falhas de segurança.
* Procure dados sensíveis enviados ao frontend.
* Verifique validação de inputs.
* Analise logs de erro.
* Procure memory leaks.
* Analise gargalos de performance.
* Identifique queries lentas no banco.

# 5. FRONTEND

* Procure componentes quebrados.
* Verifique erros de console.
* Analise renderizações desnecessárias.
* Verifique responsividade.
* Procure falhas de UX.
* Verifique loaders infinitos.
* Analise estados inconsistentes.
* Verifique formulários.
* Teste navegação entre páginas.

# 6. BANCO DE DADOS

* Verifique integridade dos dados.
* Procure duplicações.
* Verifique relacionamentos.
* Analise índices.
* Procure consultas pesadas.
* Verifique isolamento entre clientes.
* Analise risco de perda de dados.

# 7. SEGURANÇA

* Procure vulnerabilidades.
* Verifique:

  * SQL Injection
  * XSS
  * CSRF
  * Exposição de tokens
  * Chaves secretas no frontend
  * Variáveis sensíveis públicas
  * Rotas sem proteção
  * Upload inseguro
  * Escalada de privilégios
  * Vazamento entre tenants

# 8. TESTES REAIS

SIMULE:

* Usuário novo
* Usuário premium
* Usuário gratuito
* Muitos acessos simultâneos
* APIs offline
* Token expirado
* Internet lenta
* Refresh em páginas internas
* Navegação rápida entre abas

# 9. PERFORMANCE

* Identifique tudo que deixa o sistema lento.
* Verifique:

  * bundle size
  * queries pesadas
  * chamadas duplicadas
  * re-renderizações
  * cache
  * lazy loading
  * carregamento de gráficos
  * uso excessivo de CPU/RAM

# 10. RELATÓRIO FINAL

NO FINAL:

* Gere um relatório COMPLETO.
* Liste TODOS os bugs encontrados.
* Explique gravidade de cada bug.
* Mostre causa provável.
* Sugira solução ideal.
* Mostre melhorias de performance.
* Mostre melhorias de segurança.
* Priorize bugs críticos.
* NÃO QUEBRE funcionalidades existentes.
* NÃO ALTERE nada sem confirmação.
* Faça apenas análise e diagnóstico detalhado.

PRIORIDADE MÁXIMA:

* estabilidade
* segurança
* isolamento de clientes
* performance
* confiabilidade do SaaS
* experiência do usuário

Faça uma varredura extremamente profunda como uma empresa profissional de auditoria de software faria.
