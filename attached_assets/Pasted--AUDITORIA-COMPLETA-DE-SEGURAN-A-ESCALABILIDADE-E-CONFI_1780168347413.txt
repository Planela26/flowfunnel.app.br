# AUDITORIA COMPLETA DE SEGURANÇA, ESCALABILIDADE E CONFIABILIDADE DO SAAS

Atue como um Arquiteto de Software Sênior, Especialista em Cyber Security, DevSecOps, Cloud Security, Backend, Frontend, Banco de Dados e Infraestrutura.

Sua missão é realizar uma auditoria profunda de todo o sistema sem alterar funcionalidades existentes.

## Objetivo

Identificar:

* Vulnerabilidades de segurança
* Falhas de autorização
* Falhas de autenticação
* Vazamento de dados entre tenants
* Exposição de dados sensíveis
* Bugs críticos
* Falhas de escalabilidade
* Problemas de performance
* Falhas de backup e recuperação
* Possíveis vetores de ataque
* Problemas de arquitetura

Classifique cada item encontrado em:

* Crítico
* Alto
* Médio
* Baixo

Para cada problema encontrado forneça:

* Localização exata
* Impacto
* Risco
* Cenário de exploração
* Correção recomendada
* Nível de prioridade

---

# ETAPA 1 — SEGURANÇA DO FRONTEND

Auditar:

## Exposição de informações

Verificar se existem:

* API Keys expostas
* Tokens expostos
* Secrets expostos
* URLs internas
* Dados de outros usuários
* IDs previsíveis

## Segurança da aplicação

Verificar:

* XSS
* Stored XSS
* Reflected XSS
* DOM XSS
* Clickjacking
* Open Redirect
* CSRF
* Session Fixation

## Local Storage

Verificar:

* JWT armazenado no localStorage
* Tokens persistidos de forma insegura
* Dados sensíveis armazenados no navegador

## Network

Inspecionar:

* Todas as chamadas de API
* Headers enviados
* Dados trafegados
* Vazamento de informações

---

# ETAPA 2 — CONTROLE DE PERMISSÕES (RBAC)

Auditar todo o fluxo de autorização.

Verificar:

## Multi-Tenant

Confirmar que:

* Um tenant não consegue acessar dados de outro
* Todas as queries possuem filtro por tenant
* Não existe IDOR

## Usuários

Verificar:

* Admin
* Manager
* Operador
* Cliente

Testar:

* Escalada de privilégios
* Bypass de permissões
* Alteração manual de IDs
* Acesso direto por URL

## APIs

Testar:

* Todas as rotas protegidas
* Todas as rotas administrativas
* Todos os endpoints internos

Gerar relatório das rotas vulneráveis.

---

# ETAPA 3 — RATE LIMITING

Auditar proteção contra abuso.

Verificar:

## APIs

Existem limites para:

* Login
* Registro
* Recuperação de senha
* Webhooks
* Integrações
* Relatórios
* Exportações

## Ataques

Testar resistência contra:

* Brute Force
* Credential Stuffing
* Spam
* DDoS básico
* Enumeração de usuários

Identificar endpoints vulneráveis.
